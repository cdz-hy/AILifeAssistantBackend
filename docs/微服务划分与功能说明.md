**本“AI智能生活助手”共划分为以下7个微服务模块：1. 用户服务 2. 日程服务（包括天气服务）3. 财务服务 4. 饮食记录服务 5. 学习服务 6. AI分析报告服务 7. 通知服务**；以下为详细说明：

### 1. 用户服务 (User Service)



- **核心职责：** 系统的“身份与画像中心”。管理用户的“身份”、“设备”、“授权”以及由AI分析得出的“用户画像”，是所有服务的安全与个性化基石。
- **功能列表：**
  - 处理用户**注册**和**登录**，生成和验证JWT（Token）。
  - 处理用户**登出**（使Token失效）。
  - 管理用户的**个人基本资料**（如：昵称、头像）。
  - 管理用户的**数据授权**（如：是否同意系统分析健康、财务数据）。
  - 管理用户的**绑定设备**（如：手机、平板、第三方健康设备）。
  -  提供接口供AI服务**写入和更新“用户画像”**（如：作息规律、消费习惯、效率时段、拖延指数等）。
  - 提供接口供其他服务（如日程服务）**读取“用户画像”**，以实现个性化功能。
- **端口：**8001

------



### 2. 日程服务 (Schedule Service) (含天气)



- **核心职责：** 用户的“智能时间管家”。管理所有与时间相关的日程，集成天气信息，并利用AI提供决策支持。
- **功能列表：**
	- 提供日程的CRUD功能，支持丰富的属性，包括：**类型（工作/生活等）、优先级（是否紧急、是否重要）、自定义标签**。
	- 提供**日历视图**查询接口（按日、周、月查询）。
	- 管理日程的**提醒设置**。
	- 调用第三方API**查询天气**并进行数据缓存。
	- 实现天气与日程的联动，**并调用AI服务为日程提供智能建议**（如：“检测到暴雨，建议推迟该室外日程”）。
	-  (可选) 调用AI服务对用户日程安排进行**智能分析**（如合理性、冲突检测），并将结果保存。
	- **【职责核心】** 当日程提醒时间到达时，**向消息队列（MQ）发布“日程提醒”事件**。
	- **【职责核心】** 当天气突变影响日程时，**向消息队列（MQ）发布“天气预警”事件**。
	- **【职责核心】** 当日程创建或变更时，**向消息队列（MQ）发布“日程变动”事件**（携带丰富的属性和标签，供AI服务分析）。
- **端口：**8002

------



### 3. 财务服务 (Finance Service)



- **核心职责：** 用户的“智能账本”。安全地管理所有收入、支出和预算数据，并辅以AI进行智能分类。
- **功能列表：**
	- 提供**手动记账**功能（收入和支出的CRUD）。
	- (可选) **调用AI服务**对录入的交易进行**智能分类和打标签**。
	- 管理**月度/年度预算**（设置、查询、更新）。
	- 提供基础的**收支统计报表**接口（如按分类、按时间）。
	- **【职责核心】** 当支出超支时，**向消息队列（MQ）发布“预算预警”事件**。
	- **【职责核心】** 当新交易产生时，**向消息队列（MQ）发布“财务变动”事件**（供AI服务分析）。
- **端口：**8003

------



### 4. 饮食记录服务 (Diet Record Service)



- **核心职责：** 用户的“健康日志”。管理所有饮食记录和基础营养数据。
- **功能列表：**
	- 提供**记录饮食**功能（支持文字输入或图片上传）。
	- (可选) 调用AI服务查询**食物热量与营养成分**。
	- (可选) 调用AI服务实现**食物图片识别**。
	- 统计每日的**热量和营养素摄入**（蛋白质、脂肪等）。
	- **【职责核心】** 当用户记录新饮食时，**向消息队列（MQ）发布“饮食日志”事件**（供AI服务分析）。
- **端口：**8004

------



### 5. 学习服务 (Study Service)



- **核心职责：** 用户的“学习助手”。管理学习计划和专注状态。
- **功能列表：**
	- 提供**学习计划**的CRUD功能。
	- (可选) 调用AI服务**智能生成学习计划**。
	- 提供**专注模式（番茄钟）**功能（开始、暂停、停止计时）。
	- 统计**学习时长**和**计划完成度**。
	- **【职责核心】** 当专注开始/结束时，**向消息队列（MQ）发布“专注状态”事件**（供通知服务和AI服务）。
- **端口：**8005

------



### 6. AI分析报告服务 (AI Analysis Report Service)



- **核心职责：** 系统的“智能大脑”。**不拥有**原始数据，而是**消费**所有服务的数据，进行分析、建模并提供智能。
- **功能列表：**
	- **【数据输入】** **订阅**来自其他所有服务的事件（财务、饮食、学习、日程、设备健康等）。
	- **【核心分析】** **【新增】** 跨域分析所有订阅数据，**构建并动态更新“用户画像”**（如效率时段、拖延指数、健康状况），并**调用用户服务接口将其持久化**。
	- **【数据输出】** 定期（或按需）**生成综合生活分析报告**。
	- **【服务能力】** 向其他服务提供“智能计算”API，包括：**财务智能分类**、**日程分析与建议**、学习计划生成、食物图片识别等。
	- **【职责核心】** 当分析发现洞察或问题时，**向消息队列（MQ）发布“智能建议”或“健康预警”事件**（如“姿势不良预警”、“近期饮食不规律”）。
- **端口：**8006

------



### 7. 通知服务 (Notification Service)



- **核心职责：** 系统的“统一出口”。**执行**所有面向用户的提醒，并实现智能的场景化（静音、振动等）。
- **功能列表：**
	- **【数据输入】** **订阅**所有需要提醒的事件（来自日程、财务、AI、学习服务）。
	- 管理用户的**通知偏好**（如“免打扰”时段、不同类型提醒的方式）。
	- 管理通知**发送渠道**（如WebSocket、APP推送、振动、铃声）。
	- **【核心逻辑】** 根据用户偏好和当前场景（如是否夜间、是否在专注模式），**决策**最终的提醒方式（例如：将夜间的“日程提醒”铃声降级为“柔和振动”）。
	- 提供**历史通知**的查询列表。
- **端口：**8007

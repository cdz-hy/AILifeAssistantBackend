## AI生活助手：数据库设计

## 1. 用户服务 (User Service)

### **`t_user` (用户核心表)**

- `id` (PK, BIGINT): 唯一主键，用户ID。
- `username` (VARCHAR, UNIQUE): 登录用户名，必须唯一。
- `password_hash` (VARCHAR): 加密后的密码哈希值。
- `email` (VARCHAR, UNIQUE, NULLABLE): 电子邮箱，唯一，允许为空。
- `phone` (VARCHAR, UNIQUE, NULLABLE): 手机号码，唯一，允许为空。

### **`t_user_profile` (用户资料表)**

- `id` (PK, BIGINT): 唯一主键。
- `user_id` (FK -> `t_user.id`, UNIQUE): 关联的用户ID，与`t_user`表一对一。
- `nickname` (VARCHAR): 用户昵称。
- `avatar_url` (VARCHAR): 用户头像图片的URL链接。

### **`t_bound_device` (绑定设备表)**

- `id` (PK, BIGINT): 唯一主键。
- `user_id` (FK -> `t_user.id`): 关联的用户ID。
- `device_type` (VARCHAR): 设备类型 (例如: 'ios', 'android', 'web')。
- `device_token` (VARCHAR): 设备的推送令牌 (用于APNs, FCM等)。

### **`t_data_authorization` (数据授权表)**

- `id` (PK, BIGINT): 唯一主键。
- `user_id` (FK -> `t_user.id`): 关联的用户ID。
- `data_type` (VARCHAR): 授权的数据类型 (例如: 'healthkit', 'google_calendar')。
- `is_authorized` (BOOLEAN): 用户是否已授权。

### **`t_user_persona_metric` (用户画像指标表)**

- `id` (PK, BIGINT): 唯一主键。
- `user_id` (FK -> `t_user.id`): 关联的用户ID。
- `metric_name` (VARCHAR): 指标名称 (Key)，(例如: "procrastination_index", "sleep_score")。
- `metric_value` (VARCHAR 或 DECIMAL): 指标的值 (Value)，(例如: "high", 8.5)。
- `last_updated_by` (VARCHAR): 最后更新此指标的来源 (例如: 'ai_service', 'user_input')。
- `updated_at` (TIMESTAMP): 该指标的最后更新时间。
- *(约束: UNIQUE (`user_id`, `metric_name`))*

### **`t_persona_tag` (画像标签(Lookup))**

- `id` (PK, BIGINT): 标签唯一主键。
- `tag_name` (VARCHAR, UNIQUE): 标签名称 (例如: "学生", "拖延", "早睡者")。

### **`t_user_persona_tag_map` (用户-画像标签-映射表)**

- `user_id` (FK -> `t_user.id`): 关联的用户ID。
- `tag_id` (FK -> `t_persona_tag.id`): 关联的标签ID。

## 2. 日程服务 (Schedule Service)

### **`t_schedule` (日程主表)**

- `id` (PK, BIGINT): 日程的唯一主键。
- `user_id` (BIGINT): 关联的用户ID。
- `title` (VARCHAR): 日程标题。
- `description` (TEXT): 日程的详细描述。
- `start_time` (TIMESTAMP): 日程开始时间。对于重复事件，这是“首次”开始时间。
- `end_time` (TIMESTAMP): 日程结束时间。
- `type_id` (FK -> `t_schedule_type.id`): 关联的日程类型ID。
- `is_urgent` (BOOLEAN): 是否紧急 (四象限)。
- `is_important` (BOOLEAN): 是否重要 (四象限)。
- `recurrence_rule` (VARCHAR): iCal RRULE重复规则字符串 (例如: "FREQ=WEEKLY;BYDAY=MO")。
- `status` (VARCHAR): 日程状态 (例如: 'pending', 'completed')。

### **`t_schedule_exception` (日程例外表)**

- `id` (PK, BIGINT): 例外规则的唯一主键。
- `schedule_id` (FK -> `t_schedule.id`): 关联的“主”日程ID。
- `original_start_time` (TIMESTAMP): 指向此例外所要覆盖的原始（重复）日程的开始时间。
- `is_cancelled` (BOOLEAN): 是否仅取消这一次。`true`表示取消，`false`结合新时间表示修改。
- `new_title` (VARCHAR, NULLABLE): 如果修改了单次日程的标题，则存储在此。
- `new_start_time` (TIMESTAMP, NULLABLE): 如果修改了单次日程的开始时间。
- `new_end_time` (TIMESTAMP, NULLABLE): 如果修改了单次日程的结束时间。

### **`t_schedule_type` (日程类型表 (Lookup))**

- `id` (PK, BIGINT): 类型的唯一主键。
- `user_id` (BIGINT, NULL为系统预设): 关联的用户ID。NULL表示系统预设类型 (如 "工作", "生活")。
- `type_name` (VARCHAR): 类型名称。
- `color_hex` (VARCHAR): 用于前端显示的颜色代码 (例如: "#FF5733")。

### **`t_schedule_tag` (日程标签表 (Lookup))**

- `id` (PK, BIGINT): 标签的唯一主键。
- `tag_name` (VARCHAR): 标签名称。
- `created_by` (VARCHAR): 创建来源 (例如: 'user', 'ai')。

### **`t_schedule_tag_map` (日程-标签-映射表)**

- `schedule_id` (FK -> `t_schedule.id`): 关联的日程ID。
- `tag_id` (FK -> `t_schedule_tag.id`): 关联的标签ID。

### **`t_schedule_reminder` (日程提醒表)**

- `id` (PK, BIGINT): 提醒的唯一主键。
- `schedule_id` (FK -> `t_schedule.id`): 关联的日程ID。
- `remind_at` (TIMESTAMP): 提醒任务触发的绝对时间点。
- `status` (VARCHAR): 提醒的状态 (例如: 'pending', 'sent', 'failed')。

### **`t_schedule_ai_suggestion` (AI建议存储表)**

- `id` (PK, BIGINT): 建议的唯一主键。
- `schedule_id` (FK -> `t_schedule.id`): 关联的日程ID。
- `suggestion_type` (VARCHAR): 建议类型 (例如: 'time_conflict', 'reschedule_option')。
- `suggestion_content` (TEXT): 建议的具体内容。
- `is_accepted` (BOOLEAN, NULLABLE): 用户是否接受了该建议 (NULL表示未操作)。

### **`t_weather_cache` (天气缓存表)**

- `cache_key` (VARCHAR, PK): 缓存键 (例如: 'location_date' 或 'city_id_date')。
- `data_json` (JSONB): 存储的天气数据 (JSON格式)。
- `expires_at` (TIMESTAMP): 缓存的过期时间。

## 3. 财务服务 (Finance Service)

### **`t_transaction` (交易流水表)**

- `id` (PK, BIGINT): 交易流水的唯一主键。
- `user_id` (BIGINT): 关联的用户ID。
- `amount` (DECIMAL(10, 2)): 交易金额。
- `type` (VARCHAR): 交易类型 (例如: 'income', 'expense')。
- `transaction_date` (TIMESTAMP): 交易发生的时间。
- `description` (TEXT): 交易描述 (例如: "星巴克拿铁")。
- `category_id` (FK -> `t_finance_category.id`): 关联的财务分类ID。
- `currency_code` (VARCHAR(3)): 货币代码 (例如: 'CNY', 'USD')。
- `ai_suggestion_json` (JSONB): 存储AI的建议 (例如: `{"confidence": 0.9, "candidates": ["food", "drink"]}`)。

### **`t_finance_category` (财务分类表 (Lookup))**

- `id` (PK, BIGINT): 分类的唯一主键。
- `parent_id` (FK, NULLABLE): 父分类ID，用于支持多级分类 (例如: "餐饮" -> "饮品")。
- `user_id` (BIGINT, NULL为系统预设): 关联的用户ID。NULL表示系统预设分类。
- `category_name` (VARCHAR): 分类名称 (例如: "餐饮", "交通", "娱乐")。

### **`t_budget` (预算表)**

- `id` (PK, BIGINT): 预算的唯一主键。
- `user_id` (BIGINT): 关联的用户ID。
- `category_id` (FK, NULLABLE): 关联的财务分类ID。NULL表示总预算。
- `amount` (DECIMAL): 预算金额。
- `budget_month` (INT): 预算对应的月份 (1-12)。
- `budget_year` (INT): 预算对应的年份 (例如: 2025)。

## 4. 饮食记录服务 (Diet Record Service)

### **`t_food_item` **

- `id` (PK, BIGINT): 食物的唯一主键。
- `food_name` (VARCHAR, UNIQUE): 食物名称 (例如: "苹果")。
- `calories_per_100g` (DECIMAL): 每100克的卡路里 (kcal)。
- `protein_per_100g` (DECIMAL): 每100克的蛋白质 (g)。
- `fat_per_100g` (DECIMAL): 每100克的脂肪 (g)。
- `carbs_per_100g` (DECIMAL): 每100克的碳水化合物 (g)。
- `created_by` (VARCHAR): 创建来源 (例如: 'system' - 系统预设, 'user' - 用户自定义)。

### **`t_diet_log` (饮食日志(餐次)表)**

- `id` (PK, BIGINT): 饮食日志（餐次）的唯一主键。
- `user_id` (BIGINT): 关联的用户ID。
- `meal_type` (VARCHAR): 餐次类型 (例如: 'breakfast', 'lunch', 'dinner', 'snack')。
- `log_time` (TIMESTAMP): 记录该餐次的时间。

### **`t_diet_log_entry` (饮食条目表)**

- `id` (PK, BIGINT): 单条食物记录的主键。
- `log_id` (FK -> `t_diet_log.id`): 关联的餐次ID。
- `image_url` (VARCHAR, NULLABLE): 用户上传的食物图片URL。
- `food_item_id` (FK -> `t_food_item.id`, NULLABLE): 关联到食物库的ID。
- `custom_food_name` (VARCHAR, NULLABLE): 如果食物库没有，用户可填自定义名称。
- `quantity_g` (DECIMAL, NULLABLE): 食用重量（克）。
- `quantity_desc` (VARCHAR, NULLABLE): 食用描述 (例如: "1个", "半碗")。
- `calories_kcal` (DECIMAL, NULLABLE): 冗余存储该条目的总卡路里，用于快速查询。

## 5. 学习服务 (Study Service)

### **`t_study_plan` (学习计划表)**

- `id` (PK, BIGINT): 学习计划的唯一主键。
- `user_id` (BIGINT): 关联的用户ID。
- `title` (VARCHAR): 计划标题 (例如: "PMP考试冲刺")。
- `goal` (TEXT): 计划的最终目标描述。
- `start_date` (DATE): 计划开始日期。
- `end_date` (DATE): 计划结束日期。
- `status` (VARCHAR): 计划状态 (例如: 'active', 'completed', 'archived')。

### **`t_study_task` (学习任务表)**

- `id` (PK, BIGINT): 学习任务的唯一主键。
- `plan_id` (FK, NULLABLE): 关联的学习计划ID (允许任务独立存在)。
- `title` (VARCHAR): 任务标题 (例如: "完成第一章练习")。
- `is_completed` (BOOLEAN): 任务是否已完成。
- `due_date` (DATE): 任务的截止日期。

### **`t_focus_session` (专注记录表)**

- `id` (PK, BIGINT): 专注时段的唯一主键 (例如: 一次番茄钟)。
- `user_id` (BIGINT): 关联的用户ID。
- `start_time` (TIMESTAMP): 专注开始的时间。
- `duration_minutes` (INT): 实际专注的时长（分钟）。
- `status` (VARCHAR): 专注状态 (例如: 'completed', 'interrupted')。
- `related_task_id` (FK, NULLABLE): 关联的学习任务ID。

## 6. AI分析报告服务 (AI Analysis Service)

### **`t_generated_report` (分析报告存储表)**

- `id` (PK, BIGINT): 报告的唯一主键。
- `user_id` (BIGINT): 关联的用户ID。
- `report_type` (VARCHAR): 报告类型 (例如: 'daily', 'weekly', 'monthly_summary')。
- `start_date` (DATE): 报告所涵盖的开始日期。
- `end_date` (DATE): 报告所涵盖的结束日期。
- `report_data_json` (JSONB): 报告的详细内容 (AI分析结论、图表数据等) (JSON格式)。

### **`t_agg_daily_metrics` (每日聚合指标表)**

- `id` (PK, BIGINT): 唯一主键。
- `user_id` (BIGINT): 关联的用户ID。
- `date` (DATE): 指标对应的日期。
- `total_calories` (DECIMAL): 当日总摄入卡路里。
- `total_focus_minutes` (INT): 当日总专注分钟。
- `total_expense` (DECIMAL): 当日总支出。
- *(约束: UNIQUE (`user_id`, `date`))*

### **`t_ai_model_info` (AI模型信息表)**

- `id` (PK, BIGINT): 模型的唯一主键。
- `model_name` (VARCHAR): AI模型的名称。
- `version` (VARCHAR): 模型的版本号 (用于追踪报告是由哪个版本的模型生成的)。

## 7. 通知服务 (Notification Service)

### **`t_notification_log` (通知发送日志)**

- `id` (PK, BIGINT): 日志的唯一主键。
- `user_id` (BIGINT): 接收通知的用户ID。
- `content` (TEXT): 通知的具体内容。
- `channel` (VARCHAR): 实际发送的渠道 (例如: 'push', 'websocket', 'sms')。
- `status` (VARCHAR): 发送状态 (例如: 'sent', 'failed', 'read')。
- `source_event_type` (VARCHAR): 触发此通知的源事件类型 (例如: 'schedule_reminder')。
- `created_at` (TIMESTAMP): 通知创建的时间。

### **`t_notification_preference` (通知偏好表)**

- `id` (PK, BIGINT): 偏好设置的唯一主键。
- `user_id` (BIGINT): 关联的用户ID。
- `event_type` (VARCHAR): 事件类型 (例如: 'schedule_reminder', 'budget_alert', 'ai_suggestion')。
- `channel` (VARCHAR): 通知渠道 (例如: 'push', 'websocket')。
- `alert_type` (VARCHAR): 提醒方式 (例如: 'sound_and_vibration', 'vibration_only', 'silent')。
- `is_enabled` (BOOLEAN): 是否启用此条偏好。
- *(约束: UNIQUE (`user_id`, `event_type`, `channel`))*

### **`t_notification_dnd` (免打扰(DND)设置)**

- `id` (PK, BIGINT): 唯一主键。
- `user_id` (BIGINT, UNIQUE): 关联的用户ID (每个用户仅一条记录)。
- `is_enabled` (BOOLEAN): 是否开启免打扰模式。
- `start_time` (TIME): 免打扰开始时间 (时:分:秒)。
- `end_time` (TIME): 免打扰结束时间 (时:分:秒)。
